# Recommended Repository Labels

To keep the repository organized, we use the following standard labels for Issues and Pull Requests.

## Issue Types
- `bug`: Something isn't working as expected.
- `enhancement`: New feature or request.
- `documentation`: Improvements or additions to documentation.
- `question`: Further information is requested.
- `discussion`: Requires community or maintainer discussion.

## Subsystems
- `engine`: Relates to the core execution engine (Downloader, Installer, Resolver).
- `registry`: Relates to the static registry JSON, builder, or provider.
- `cli`: Relates to the developer command-line interface.
- `components`: Relates to the UI component library.
- `framework`: Relates to UI frameworks (e.g. CustomTkinter).

## Triage & Effort
- `good first issue`: Good for newcomers.
- `help wanted`: Extra attention is needed.
- `priority: high`: Urgent fixes, blocking bugs.
- `priority: medium`: Standard feature work.
- `priority: low`: Nice to have, low impact.
