# LifeOfPy Installer Engine

The final architectural layer of the LifeOfPy engine. The Installer acts exclusively as an orchestrator, consuming the `InstallPlan` generated by the Dependency Graph, triggering the `Downloader`, mapping the staging directories to the `Validator`, and finally executing atomic transactions to write to the user's project workspace.

## Core Features
1. **Atomic Transactions (`transactions.py`)**: Uses a strict Project Lock `.lifeofpy.lock.pid` to ensure a deterministic environment, preventing concurrent CLI interference.
2. **Strict Verification (`verification.py`)**: Verifies the final staged artifact dynamically via the `ManifestValidatorService`.
3. **Rollback Safety (`rollback.py`)**: Any failure (network drop, hash mismatch, syntax error) mid-installation triggers an immediate unroll of all applied target directories back to their initial state.
4. **State Management (`history.py`, `lockfile.py`)**: Mutates the `.lifeofpy/history.json` and `lifeofpy.lock` structures immediately upon atomic commit, guaranteeing repeatable tracking.

## Usage

```python
from engine.installer.service import InstallerService

installer = InstallerService(logger, fs, downloader, validator)

# Bind the UI
installer.subscribe(lambda event: print(f"{event.name}: {event.component_slug}"))

# Execute the DAG safely
installer.install(install_plan, Path("/my/project"))
```
