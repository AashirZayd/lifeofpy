# Security Policy

## Supported Versions

LifeOfPy is currently in its early stages (Alpha). Only the latest release on the `main` branch is actively supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |
| < 0.1   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in LifeOfPy (CLI, Engine, or Registry), please do NOT report it by creating a public GitHub issue.

Instead, please send an email to **security@lifeofpy.com** (placeholder) or use GitHub Security Advisories to privately report the vulnerability.

### What to include
- A description of the vulnerability.
- Steps to reproduce the issue.
- Potential impact and affected subsystems.

### Expected Response Time
We take security seriously. You should expect an initial acknowledgement of your report within 48 hours. We will keep you updated on the progress of the investigation and the timeline for a patch.

## Security Disclosure Policy

1. Once a vulnerability is verified, we will work to patch the issue promptly.
2. We will release a security patch and issue a CVE if appropriate.
3. We will publicly disclose the vulnerability details only after the patch has been released and users have been notified to upgrade.
